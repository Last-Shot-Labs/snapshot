# i18n

`manifest.i18n` declares the app locale list, the default locale, and the translation strings used by `{ "t": "..." }` references.

## Shape

```json
{
  "i18n": {
    "default": "en",
    "locales": ["en", "es"],
    "detect": ["state", "navigator", "default", "header"],
    "strings": {
      "en": {
        "auth": {
          "loginTitle": "Sign in"
        }
      },
      "es": {
        "auth": {
          "loginTitle": "Iniciar sesiĂ³n"
        }
      }
    }
  }
}
```

## Detection order

`detect` is evaluated in order. The first matching locale wins.

- `state`: reads `manifest.state.locale`
- `navigator`: uses `navigator.language`
- `default`: uses `manifest.i18n.default`
- `header`: uses `Accept-Language` on the server

## Translation refs

Text-bearing manifest fields can use:

```json
{ "t": "auth.loginTitle" }
```

You can also provide variables:

```json
{ "t": "greeting", "vars": { "name": "Ada" } }
```

## Notes

- Missing keys fall back to the default locale, then to the key name.
- Locale persistence uses the normal `manifest.state.locale` path.
