# Snapshot Playground

The playground is a canonical showcase surface, not disposable demo code.

## Read First

- `playground/src/showcase.tsx`
- `playground/src/app.tsx`
- `playground/src/token-editor.tsx`
- `playground/src/styles.css`
- `src/ui.ts`

## Expectations

- New user-facing components should be represented in the playground unless there is a clear reason not to.
- Showcase states should be realistic enough to support documentation screenshots, examples, and validation.
- If a guide links to the playground, the showcased behavior must reflect current source.

## Required Follow-Through

- keep showcase metadata in sync with the component catalog
- update docs links when sections or pages move
- do not add placeholder demos that do not reflect real config/runtime behavior
